# Frontend Handover: Courses API Integration

## Audience

Frontend engineers integrating the courses catalog, course details, enrollment, and subscription-status flows.

## Service Overview

- Base URL (local): `http://localhost:3001`
- Resource root: `/courses`
- Validation: Global validation pipe is enabled with transform + whitelist + forbidNonWhitelisted.
- Error contract: Standardized (see Error Handling section).
- Request tracing: `X-Request-Id` is accepted from client or generated by backend and returned in response headers.

## Authentication Model (Current vs Target)

### Current state in this repository

`JwtAuthGuard`, `OptionalJwtAuthGuard`, and `RolesGuard` are placeholder guards and currently return `true`.

### Target/expected frontend behavior

Even though guards are permissive today, frontend should integrate as if auth is enforced:

- Include `Authorization: Bearer <token>` for protected endpoints.
- Handle `401/403` responses in the API client (future-ready).
- Do not hard-code assumptions based on current permissive behavior.

## Endpoints

## 1) List all active courses

- Method: `GET`
- Path: `/courses`
- Auth: Optional
- Cache interceptor: Enabled

### Query params

- `page` (optional, number, min 1, default 1)
- `limit` (optional, number, min 1, default 3)
- `category` (optional, string)

### Example

```http
GET /courses?page=1&limit=6&category=Featured
```

### Response shape

```json
{
  "items": [
    {
      "id": "uuid",
      "title": "Course Title",
      "description": "...",
      "category": "Featured",
      "status": "active",
      "imageUrl": "https://...",
      "currentPrice": 0,
      "thumbnail": "https://...",
      "price": 0,
      "isPublished": true,
      "instructor": {
        "id": "uuid",
        "name": "Instructor Name",
        "avatar": "https://...",
        "title": "Senior Mentor"
      },
      "isSubscribed": false

  "pagination": {
    "currentPage": 1,
    "totalPages": 5,
    "totalCourses": 29,
    "hasNextPage": true,
    "hasPreviousPage": false
  }
}
```

- Method: `GET`
- Path: `/courses/scholarx`
- Auth: Optional
- Cache interceptor: Enabled

### Query params

- `page` (optional)

### Notes

Uses category filter `ScholarX` server-side.

## 4) Search courses

- Method: `GET`
- Path: `/courses/search`
- Auth: Optional
- Cache interceptor: Enabled

### Query params

- `title` (required in practice; if missing server returns 400)
- `page` (optional)
- `limit` (optional)

### Example

```http
GET /courses/search?title=react&page=1&limit=12
```

## 5) Get course details

- Method: `GET`
- Path: `/courses/:id`
- Auth: Optional

### Path params

- `id` (required UUID)

### Response

Single mapped course object with `isSubscribed` computed when user context exists.

## 6) Check current user subscription status for a course

- Method: `GET`
- Path: `/courses/:id/subscription-status`
- Auth: Required

### Path params

- `id` (required UUID)

### Response

```json
{
  "isSubscribed": true,
  "courseId": "uuid",
  "userId": "uuid"
}
```

## 7) Enroll current user to course

- Method: `POST`
- Path: `/courses/:id/enroll`
- Auth: Required

### Path params

- `id` (required UUID)

### Success response

```json
{
  "course": {
    "id": "uuid",
    "studentsCount": 101
  },
  "userId": "uuid"
}
```

### Common business errors

- 404: Course not found
- 404: User not found
- 403: User is blocked
- 400: Already enrolled

## 8) Create course (admin flow)

- Method: `POST`
- Path: `/courses`
- Auth: Required (plus role guard)
- Multipart: Supported (`image` file field)

### Body fields

- `title` (string, required, 3..100)
- `description` (string, required, 10..2000)
- `category` (enum: `Featured` | `ScholarX`, required)
- `currentPrice` (number, min 0, required)
- `originalPrice` (number, optional)
- `slug` (string, optional)
- `videoPreviewUrl` (string, optional)
- `isBestseller` (boolean, optional)
- `urgencyText` (string, optional)
- `tags` (string[], optional)
- `level` (enum: `Beginner` | `Intermediate` | `Advanced`, optional)
- `duration` (string, optional)
- `instructorId` (UUID, optional)
- `status` (enum: `active` | `inactive`, optional)
- `image` (file, optional, multipart field name must be exactly `image`)

## 9) Update course (admin flow)

- Method: `PUT`
- Path: `/courses/:id`
- Auth: Required (plus role guard)
- Multipart: Supported (`image` optional)

- Method: `PATCH`
- Path: `/courses/:id`
- Auth: Required (plus role guard)
- Multipart: Supported (`image` optional)

### Notes

- `PUT` and `PATCH` currently route to the same partial update behavior.
- Use `PATCH` for forward compatibility.

## 10) Delete course (soft delete)

- Method: `DELETE`
- Path: `/courses/:id`
- Auth: Required (plus role guard)

### Behavior

Soft delete by setting `status = inactive`.

## Error Handling Contract

All errors are returned as:

```json
{
  "success": false,
  "error": {
    "statusCode": 422,
    "code": "VALIDATION_ERROR",
    "numericCode": 1000,
    "message": "Validation failed",
    "details": [
      {
        "field": "title",
        "constraints": {
          "isString": "title must be a string"
        }
      }
    ]
  },
  "meta": {
    "timestamp": "2026-03-15T10:00:00.000Z",
    "path": "/courses/search",
    "method": "GET",
    "requestId": "uuid-or-no-request-id"
  }
}
```

## Frontend Integration Notes

- Send `X-Request-Id` from frontend if you want full cross-service traceability.
- Parse and store `meta.requestId` for support tooling.
- Prefer `thumbnail`, `price`, and `isPublished` aliases in UI code for consistency.
- For enrollment CTA logic, use `GET /courses/:id/subscription-status` as source of truth.
- After successful enroll, revalidate/refetch:
  - course details (`/courses/:id`)
  - listing page currently in view (`/courses`, `/featured`, or `/scholarx`)

## Caching Caveat

List/search/featured/scholarx routes use `CacheInterceptor` and optional user context.
If per-user subscription state appears stale in list cards, refresh using:

1. A details fetch (`GET /courses/:id`) or
2. Direct status check (`GET /courses/:id/subscription-status`)

until cache keying is explicitly made user-aware.

## Example Fetch Wrappers

```ts
const API_BASE = process.env.NEXT_PUBLIC_API_BASE_URL ?? 'http://localhost:3001';

export async function fetchCourses(params: {
  page?: number;
  limit?: number;
  category?: string;
  token?: string;
}): Promise<PaginatedCoursesResponse> {
  const url = new URL('/courses', API_BASE);
  if (params.page) url.searchParams.set('page', String(params.page));
  if (params.limit) url.searchParams.set('limit', String(params.limit));
  if (params.category) url.searchParams.set('category', params.category);

  const res = await fetch(url, {
    headers: {
      ...(params.token ? { Authorization: `Bearer ${params.token}` } : {}),
      'Content-Type': 'application/json',
      'X-Request-Id': crypto.randomUUID(),
    },
    cache: 'no-store',
  });

  if (!res.ok) throw await res.json();
  return res.json();
}

export async function enrollCourse(courseId: string, token: string) {
  const res = await fetch(`${API_BASE}/courses/${courseId}/enroll`, {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${token}`,
      'X-Request-Id': crypto.randomUUID(),
    },
  });

  if (!res.ok) throw await res.json();
  return res.json();
}
```

## QA Checklist for Frontend

- Verify pagination controls with `page` and `limit`.
- Verify search requires non-empty `title` query.
- Verify graceful handling for UUID validation errors on `:id` routes.
- Verify enroll duplicate attempt returns 400 and shows proper message.
- Verify blocked-user scenario returns 403.
- Verify subscription badge logic after enroll with forced revalidation.
